# 기술 지원 문의 응대 가이드

재현 가능한 상세 정보 수집, 스크린샷/로그 파일 요청, 시스템 환경 확인을 우선으로 합니다.

## 로그인 오류

**문의**: 로그인이 안 돼요

**답변 방법**:
- 오류 메시지 정확히 파악
- 브라우저 캐시/쿠키 삭제 안내
- 다른 브라우저나 시크릿 모드 테스트 권장
- 비밀번호 재설정 안내

**답변 예시**:
"로그인 오류 해결을 위해 1) 브라우저 캐시 및 쿠키 삭제, 2) 시크릿/프라이빗 모드에서 재시도, 3) 다른 브라우저 테스트를 권장합니다. 문제가 지속되면 오류 메시지 스크린샷과 브라우저 콘솔 로그(F12 → Console)를 회신해 주시기 바랍니다."

## 기능 작동 오류

**문의**: 특정 기능이 작동하지 않아요

**답변 방법**:
- 재현 단계 상세히 수집 (어떤 작업 → 예상 결과 → 실제 결과)
- 시스템 정보 확인 (브라우저, OS, 버전)
- 알려진 버그 확인
- 임시 해결책 제시

**답변 예시**:
"기능 오류를 해결하기 위해 재현 단계를 알려주시기 바랍니다: 1) 어떤 작업을 하셨나요? 2) 예상 결과는? 3) 실제로 어떻게 동작했나요? 브라우저 버전과 OS 정보도 함께 회신해 주시면 개발팀에 전달하겠습니다."

## 성능 저하/느림

**문의**: 서비스가 너무 느려요

**답변 방법**:
- 시스템 상태 페이지 확인 안내
- 네트워크 속도 테스트 권장
- 브라우저 확장 프로그램 비활성화 제안
- 데이터량/복잡도 확인

**답변 예시**:
"성능 저하 원인 파악을 위해 1) 시스템 상태 페이지 [URL] 확인, 2) 인터넷 속도 테스트(fast.com), 3) 브라우저 확장 프로그램 비활성화 후 재시도를 권장합니다. 특정 작업에서만 느리다면 작업 내용과 처리 중인 데이터량을 알려주시기 바랍니다."

## 데이터 동기화 문제

**문의**: 데이터가 동기화되지 않아요

**답변 방법**:
- 마지막 동기화 시간 확인 안내
- 수동 동기화 시도 권장
- 네트워크 상태 확인
- 동기화 로그 확인 요청

**답변 예시**:
"동기화 문제 해결을 위해 1) 설정 > 동기화 > 마지막 동기화 시간 확인, 2) 수동 동기화 버튼 클릭, 3) 네트워크 연결 확인을 시도해 주시기 바랍니다. 문제가 지속되면 설정 > 고급 > 동기화 로그를 다운로드하여 회신해 주시기 바랍니다."

## API/통합 오류

**문의**: API 연동이 안 돼요

**답변 방법**:
- API 키 유효성 확인
- 오류 코드 해석 (400, 401, 429, 500 등)
- Rate limit 확인
- API 문서 참조 안내

**답변 예시**:
"API 오류 코드에 따른 해결 방법: 400(요청 형식 오류) → 필수 파라미터 확인, 401(인증 실패) → API 키 재발급, 429(요청 초과) → Rate limit 확인 [플랜별 한도], 500(서버 오류) → 당사 조치 필요. API 문서 [URL]를 참고하시고, 요청 예시를 회신해 주시면 상세 분석하겠습니다."

## 모바일 앱 크래시

**문의**: 앱이 자꾸 종료돼요

**답변 방법**:
- 앱 버전 및 디바이스 정보 수집
- 앱 강제 종료 후 재시작 안내
- 앱 업데이트 확인
- 재설치 권장 (백업 후)

**답변 예시**:
"앱 크래시 해결을 위해 1) 앱 강제 종료 후 재시작, 2) 앱 최신 버전 확인 및 업데이트, 3) 디바이스 재시작, 4) 앱 재설치(데이터 백업 후)를 시도해 주시기 바랍니다. 앱 버전과 디바이스 모델, 크래시 발생 시점을 회신해 주시면 추가 조치하겠습니다."

## 데이터 손실/복구

**문의**: 데이터가 사라졌어요

**답변 방법**:
- 백업 상태 즉시 확인
- 복구 가능 범위 안내 (자동 백업 30일, 버전 히스토리)
- 휴지통 확인 권장
- 긴급 처리 우선

**답변 예시**:
"데이터 복구를 위해 1) 휴지통 확인(삭제 후 30일 보관), 2) 설정 > 백업에서 복구 가능 시점 확인, 3) 파일별 버전 히스토리 복구를 시도해 주시기 바랍니다. 복구 불가 시 개발팀 데이터베이스 레벨 복구를 시도하며 48시간 소요됩니다. 추가 작업을 중단하여 덮어쓰기를 방지해 주시기 바랍니다."

## 이메일/알림 미수신

**문의**: 이메일이나 알림을 못 받았어요

**답변 방법**:
- 스팸 폴더 확인 권장
- 이메일 주소 정확성 확인
- 알림 설정 확인 안내
- 이메일 재전송 처리

**답변 예시**:
"이메일 미수신 시 1) 스팸 폴더 확인, 2) 고객 포털에서 등록된 이메일 주소 확인, 3) 알림 설정(설정 > 알림) 확인을 권장합니다. 등록된 이메일 주소를 회신해 주시면 이메일을 재전송해 드리겠습니다."
